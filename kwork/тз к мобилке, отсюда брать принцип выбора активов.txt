Общие требования
Мобильное приложение должно предоставлять пользователю доступ к мониторингу финансовых активов, включая акции, валютные пары и криптовалюты. Приложение должно поддерживать добавление активов в избранное для быстрого доступа.

Если что то не описано, все на ваш выбор. Главное реализовать это:

Дизайн:
- Минимализм. Цветовая гамма на твой выбор
- Анимации при переходах
- Кнопка назад чтобы возращатся на предыдущий экран

Основные вкладки
После запуска пользователь видит четыре вкладки:

Избранные активы: включает список избранных активов пользователя, добавленных из других вкладок.
Акции: предоставляет доступ к акциям по странам, а также данные по графику и текущим показателям акций.
Валюты: доступ к валютным парам с возможностью конструктора пар (например, USD/RUB).
Криптовалюты: отображает список всех криптовалют с возможностью просмотра данных по выбранной криптовалюте.
Функционал
Поиск: На каждой вкладке предусмотрена строка поиска для быстрого нахождения активов по имени или тикеру.
Добавление в избранное: У каждого актива есть кнопка «Добавить в избранное», позволяющая добавлять актив в избранное для быстрого доступа.
После выбора актива у пользователя будет краткая сводка по активу и график. Выбор графика может быть 1h 1d 1w

Вкладка "Акции"
Шаг 1: Пользователь выбирает страну из списка (API предоставляет список стран с ключом country).
Шаг 2: После выбора страны пользователь видит список акций с информацией name (имя) и symbol (краткое обозначение).
Шаг 3: При выборе акции загружается:
График исторических данных – отображает данные о ценах актива за определенный период.
Текущие показатели актива – отображаются актуальные данные, такие как текущая цена, объем, дневные изменения и т.п.
Эндпоинты для вкладки "Акции":
Получение списка акций по стране: GET /stocks?country={country}
Исторические данные по акции: GET /historical/stock/{symbol}
Текущие данные по акции: GET /current/stock/{symbol}

Вкладка "Валюты"
Шаг 1: Пользователь выбирает валютные пары, используя конструктор (например, выбор пары USD/RUB).
Шаг 2: Приложение загружает и отображает:
График исторических данных – для выбранной пары валют.
Текущие показатели пары валют – отображаются актуальные данные, такие как текущий курс, объем и дневные изменения.
Эндпоинты для вкладки "Валюты":
Получение списка валютных пар: GET /currency_crosses
Исторические данные по паре валют: GET /historical/currency/{symbol}
Текущие данные по паре валют: GET /current/currency/{symbol}

Вкладка "Криптовалюты"
Шаг 1: Пользователь видит список всех доступных криптовалют (например, BTC, ETH, XRP).
Шаг 2: При выборе криптовалюты приложение загружает:
График исторических данных – для выбранной криптовалюты.
Текущие показатели криптовалюты – такие как текущая цена, объем и дневные изменения.
Эндпоинты для вкладки "Криптовалюты":
Получение списка криптовалют: GET /cryptocurrencies
Исторические данные по криптовалюте: GET /historical/crypto/{symbol}
Текущие данные по криптовалюте: GET /current/crypto/{symbol}

Примеры конечных адресов
### Примеры конечных точек API для мобильного приложения

#### Список активов по категориям

1. **Список акций**
   ```http
   GET http://bigidulka2.ddns.net:8000/stocks
   ```

2. **Список криптовалют**
   ```http
   GET http://bigidulka2.ddns.net:8000/cryptocurrencies
   ```

3. **Список валютных пар**
   ```http
   GET http://bigidulka2.ddns.net:8000/currency_crosses
   ```

#### Текущие данные для актива

1. **Текущие данные для криптовалюты BTC**
   ```http
   GET http://bigidulka2.ddns.net:8000/current/crypto/BTC?interval=1d
   ```

2. **Текущие данные для акции AAPL**
   ```http
   GET http://bigidulka2.ddns.net:8000/current/stock/AAPL?interval=1d
   ```

3. **Текущие данные для валюты RUB**
   ```http
   GET http://bigidulka2.ddns.net:8000/current/currency/RUB?interval=1d
   ```

#### Исторические данные для актива

1. **Исторические данные для криптовалюты ETH (1-минутные интервалы за указанный период)**
   ```http
   GET http://bigidulka2.ddns.net:8000/historical/crypto/ETH?from_date=2024-11-09&to_date=2024-11-10&utc_offset=0&interval=1m
   ```

2. **Исторические данные для акции AAPL (1-часовые интервалы за указанный период, UTC+3)**
   ```http
   GET http://bigidulka2.ddns.net:8000/historical/stock/AAPL?from_date=2024-05-01&to_date=2024-06-01&utc_offset=3&interval=1h
   ```

3. **Исторические данные для криптовалюты BTC (дневные интервалы за указанный период, UTC-5)**
   ```http
   GET http://bigidulka2.ddns.net:8000/historical/crypto/BTC?from_date=2024-05-01&to_date=2024-06-01&utc_offset=-5&interval=1d
   ```

4. **Исторические данные для валютной пары EURUSD (дневные интервалы за указанный период, UTC-5)**
   ```http
   GET http://bigidulka2.ddns.net:8000/historical/currency/EURUSD?from_date=2024-05-01&to_date=2024-06-01&utc_offset=-5&interval=1d
   ```

5. **Исторические данные для валюты EUR (дневные интервалы за указанный период, UTC-5)**
   ```http
   GET http://bigidulka2.ddns.net:8000/historical/currency/EUR?from_date=2024-05-01&to_date=2024-06-01&utc_offset=-5&interval=1d
   ```
